netmon — сетевой монитор в стиле htop + веб-интерфейс v2.0
=========================================================

Назначение
----------
Профессиональный монитор сетевых соединений, трафика и системных
ресурсов. Доступен в двух режимах: TUI (консоль, htop-подобный)
и Web (REST API + SPA-интерфейс).

Возможности
-----------
* список слушающих TCP/UDP-серверов с PID, процессом, трафиком (RX/TX)
  и количеством клиентов;
* детализацию клиентов (включая OpenVPN-клиентов: common name, трафик,
  адрес, время подключения);
* сводка по активным пользователям, сессиям SSH и неудачным попыткам входа;
* мониторинг дисков: usage, I/O, прогноз заполнения;
* мониторинг сети: ошибки на интерфейсах, дропы, коллизии;
* детектор сканирования портов (подозрительные IP);
* iptables — показ правил файрвола, привязанных к портам;
* Nginx/Apache — парсинг access-логов (топ IP, топ путей, статусы);
* DNS — статистика запросов через systemd-resolved / BIND / tcpdump;
* Kill процесса — кнопка в веб-интерфейсе, клавиша K в TUI.

Требования
----------
* Linux
* Python 3.10+
* пакет psutil
* textual (для TUI)
* fastapi + uvicorn (для веб-сервера)
* nethogs (для per-PID трафика; если не установлен, трафик будет показывать 0)
* права root (или sudo) для доступа к сетевым соединениям и /proc

Быстрая установка
-----------------
    sudo apt install python3-pip nethogs
    pip install psutil textual fastapi uvicorn
    # или: pip install -r requirements.txt

Запуск (TUI)
------------
    sudo python -m netmon.netmon

Запуск (Web API)
----------------
    sudo python -m netmon.api_run
    # с выбором IP:
    sudo python -m netmon.api_run --ip auto
    # явно указать IP и порт:
    sudo python -m netmon.api_run --ip 192.168.1.100 --port 8000

После запуска откройте в браузере: http://IP:8000

Горячие клавиши (TUI)
---------------------
↑ / ↓              – навигация по таблице серверов
Enter              – выбрать сервер, показать его клиентов
/                  – поиск/фильтр по PID, имени или порту
ESC                – сбросить фильтр
Пробел             – пауза / возобновление автообновления (2 сек.)
S                  – показать / скрыть таблицу клиентов
K                  – kill выбранного процесса
TAB / 1-6          – переключение вкладок мониторов
D / N / P / F / W  – быстрый переход к Disk/Network/PortScan/Firewall/Web
?                  – справка
Q                  – выход

Веб-интерфейс (клавиши)
-----------------------
/                  – поиск
Space              – пауза/продолжить
S                  – показать/скрыть клиентов
K                  – kill процесса
1-6                – переключение табов мониторов
Escape             – сброс фильтра

API endpoints (все GET, кроме /api/kill)
-----------------------------------------
/api/servers       – список серверов с трафиком и клиентами
/api/login-info    – информация о входах в систему
/api/summary       – краткая сводка
/api/disk          – диски: usage, I/O, прогноз
/api/network       – сетевые интерфейсы: ошибки, дропы, коллизии
/api/port-scans    – детектор сканирования портов
/api/firewall      – правила iptables (?port=N для фильтрации)
/api/webserver     – статистика Nginx/Apache из access-логов
/api/dns           – статистика DNS-запросов
/api/all           – все данные одним запросом
POST /api/kill/{pid} – завершить процесс (?force=true для SIGKILL)

Особенности
-----------
* Для OpenVPN-серверов — подключение к management-интерфейсу.
* Права root необходимы для чтения /proc/<pid>/io и списка соединений.
* Веб-интерфейс автоматически определяет IP интерфейсов для биндинга.
* Прогноз заполнения диска — линейная экстраполяция по истории замеров.
* Порог детектора сканирования — 10 разных портов с одного IP за 60 с.
